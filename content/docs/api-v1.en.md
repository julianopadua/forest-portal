A public, read-only HTTP API for the Instituto Forest open-data catalog.

## Overview

The API serves metadata only. Dataset files are not stored by Forest and are not served by this API. Each dataset item exposes a canonical `source_url` that points to the official source. Forest pipelines may download those files temporarily during a local or cron run to validate and profile them, then publish only the manifest and profile metadata.

Base URL: `https://institutoforest.org/api/v1`

All responses are JSON, UTF-8, with the same envelope: `schema_version`, `api_version`, `generated_at`, `generation_status`, `warnings[]`, plus a payload key such as `datasets`, `manifest`, `items`, `reports`, or `sources`.

## Authentication

None. The catalog is public open data. Requests do not require credentials. CORS is permissive (`Access-Control-Allow-Origin: *`).

## Versioning

The API version is in the URL prefix (`/api/v1`). The dataset manifest has its own `schema_version`. Dataset manifests returned by this contract use schema `2.0`.

Breaking API changes require a new API path. Breaking manifest changes require a manifest schema bump.

## Storage Model

Forest stores:

- Catalog envelopes.
- Dataset manifests.
- Report manifests and report JSON.
- Compact metadata catalogs such as ANP.
- Profile metadata generated by pipeline runs.

Forest does not store raw dataset payload files in Supabase. API consumers and SDK users download bytes from item `source_url` values.

## Dataset URLs

`source_dataset_url` is the official landing page or package page for the dataset. Use it for attribution, documentation, and manual inspection.

Item `source_url` is the official downloadable resource URL. Use it for automated download.

## Profiling Contract

Profiling happens during pipeline execution. The pipeline downloads a source file to temporary local storage, inspects it, records metrics, then deletes the temporary file unless a debug flag keeps it locally.

`profiled_at` is the ISO 8601 UTC timestamp when Forest finished profiling that item. It is not the source release date and not the API response time.

`profile_status` describes the item-level profiling result:

| Value | Meaning | Consumer guidance |
|------|---------|-------------------|
| `ok` | Download succeeded, format validation passed, and expected metrics were computed. | Suitable for automated use. |
| `partial` | Download succeeded, but some metrics are incomplete because of parser limits, archive structure, unsupported members, or sampling. | Use if required fields are present. Inspect `profile_warnings`. |
| `failed` | The URL was discovered, but download, parsing, or validation failed. | Do not automate ingestion without independent validation. |
| `skipped` | Profiling was intentionally not attempted, usually because the format is unsupported or the item is not a tabular file. | Use as a link record. Do not assume row or column metrics. |

`profile_warnings` is an array of objects with `code` and `message`. Messages are public-safe and do not include stack traces, secrets, or local paths.

Common warning codes:

| Code | Meaning |
|------|---------|
| `head_unavailable` | Remote metadata could not be read through HEAD. |
| `download_timeout` | The resource could not be downloaded within the profiling timeout. |
| `unsupported_format` | No parser exists for the file format. |
| `row_count_sampled` | Row count was estimated or sampled instead of fully counted. |
| `archive_member_skipped` | Some archive members were not profiled. |
| `empty_tabular_data` | The file had no data rows or only a header row. |
| `checksum_unavailable` | A checksum was not computed or not available for this item. |

## OpenDataItem Fields

| Field | Type | Required | Description |
|------|------|----------|-------------|
| `kind` | string | yes | Currently `data`. |
| `period` | string | yes | Time partition such as `2024`, `2024-03`, an ISO date, or `Atual`. |
| `filename` | string | yes | Filename inferred from the official source URL or source metadata. |
| `source_url` | URL string | yes | Canonical official URL for downloading the resource. |
| `title` | string | no | Human-readable item title. |
| `release_time` | string | no | Publication time when the source distinguishes release slots. |
| `size_bytes` | integer | no | Bytes read during profiling. |
| `sha256` | string | no | Hash computed during profiling when the file was downloaded. |
| `row_count` | integer | no | Number of data rows when measurable. |
| `column_count` | integer | no | Number of columns when measurable. |
| `columns` | string array | no | Column names when available. |
| `content_type` | string or null | no | HTTP content type observed during profiling. |
| `format` | string | no | File format inferred from filename or source metadata. |
| `last_modified` | string or null | no | Last-Modified header observed during profiling. |
| `profiled_at` | ISO string | no | Time profiling finished. |
| `profile_status` | string | no | `ok`, `partial`, `failed`, or `skipped`. |
| `profile_warnings` | array | no | Public-safe warning objects. |
| `archive_profile` | object | no | Archive member summary for ZIP-like resources. |

## Choosing Items For Automated Use

For automated ingestion:

1. Prefer items where `profile_status` is `ok`.
2. Require `source_url`.
3. Use `sha256` verification when present.
4. Treat `row_count`, `column_count`, and `columns` as profiling metadata, not as legal guarantees from the source agency.
5. Inspect `profile_warnings` before using `partial`, `failed`, or `skipped` items.
6. Do not assume `source_dataset_url` is downloadable. It is usually a landing page.

## Endpoints

### `GET /health`

Returns service status and the manifest schema version understood by this deployment.

```bash
curl https://institutoforest.org/api/v1/health
```

### `GET /catalog`

Returns compact dataset summaries. Use `/datasets/{id}` for a full manifest.

```bash
curl https://institutoforest.org/api/v1/catalog
```

### `GET /datasets/{id}`

Returns the full dataset manifest. The path accepts dataset id or slug.

```bash
curl https://institutoforest.org/api/v1/datasets/inpe_bdqueimadas_focos
```

### `GET /datasets/{id}/items`

Returns only `items[]`. Use this when a client needs source URLs and profile metadata without the full manifest.

```bash
curl https://institutoforest.org/api/v1/datasets/inpe_bdqueimadas_focos/items
```

### `GET /catalog/reports`

Returns compact report summaries.

### `GET /reports/{id}`

Returns a report manifest. Reports may still point to derived JSON artifacts stored by Forest.

### `GET /sources`

Returns source agencies and dataset counts.

### `GET /openapi.json`

Returns the OpenAPI 3.1 document generated from the runtime Zod schemas.

## Python SDK

```bash
pip install forest-data
```

```python
import forest_data

client = forest_data.Client()
manifest = client.get_dataset("inpe_bdqueimadas_focos")

for item in manifest.items:
    print(item.period, item.source_url, item.profile_status, item.row_count)

paths = client.download("inpe_bdqueimadas_focos", path="./data")
```

The SDK follows `source_url`. If `sha256` is present, the SDK verifies the downloaded bytes by default.

## Errors

Error responses follow RFC 7807 with `Content-Type: application/problem+json`.

```json
{
  "type": "https://institutoforest.org/api/errors/not-found",
  "title": "Resource not found",
  "status": 404,
  "detail": "No dataset with id or slug \"inpe_unknown\"."
}
```
