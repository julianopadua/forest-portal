The Internet as we know it would not exist without free software. Linux runs most of the planet's servers. Git is the backbone of distributed software development. Python and R, the two most widely used languages in data science, are open projects with thousands of contributors. Before it was philosophy or business strategy, open source was a practice: publish code so anyone can read, modify, and redistribute it.

## Where the term came from

Sharing code has roots older than the internet. On the ARPANET of the 1960s, network protocols developed through public review via RFC (Request for Comments), a process that remains active today. IBM in the 1950s distributed source code alongside its operating systems. Sharing technical knowledge, in that sense, is older than proprietary software.

The term "open source" itself emerged in 1998. Christine Peterson coined the phrase during a meeting in Palo Alto, prompted by Netscape's announcement that it would release the Navigator source code. Linus Torvalds endorsed the term the next day. Eric Raymond and Bruce Perens founded the Open Source Initiative, which defines formal criteria for a license to count as open source. The event became known as the Open Source Summit, organized by O'Reilly Media, and brought together central figures of the moment: Larry Wall, Guido van Rossum, Jamie Zawinski, among others.

The term was created, in part, to separate the practice of releasing code from the "free software" movement led by Richard Stallman and the Free Software Foundation. The difference is more political than technical: Stallman treats freedom as a fundamental ethical value; the Open Source Initiative preferred a framing oriented toward collaborative development and corporate adoption. In practice, code in both cases is public and reusable.

## Software and computing: the core

In computing, the impact of open source is measurable. The Battery Open Source Software Index (BOSS) mapped, in 2017, the ten open source projects with the highest estimated economic value, considering online discussion activity, GitHub presence, search volume, and influence on the labor market:

| Rank | Project        | Associated company | Estimated value   |
|------|----------------|--------------------|-------------------|
| 1    | Linux          | Red Hat            | US$ 16 billion    |
| 2    | Git            | GitHub             | US$ 2 billion     |
| 3    | MySQL          | Oracle             | US$ 1.87 billion  |
| 4    | Node.js        | NodeSource         | not estimated     |
| 5    | Docker         | Docker             | US$ 1 billion     |
| 6    | Hadoop         | Cloudera           | US$ 3 billion     |
| 7    | Elasticsearch  | Elastic NV         | US$ 700 million   |
| 8    | Spark          | Databricks         | US$ 513 million   |
| 9    | MongoDB        | MongoDB Inc.       | US$ 1.57 billion  |
| 10   | Selenium       | Sauce Labs         | US$ 470 million   |

These projects are not academic curiosities. They are the real infrastructure on which banks, governments, scientific research, and much of cloud computing run. Linux is estimated at US$ 16 billion because the world depends on it. Hadoop and Spark underpin large-scale distributed data processing. Git and Docker are tools that any software engineer uses daily.

Three characteristics define the model in a software context. Code can be inspected and audited by anyone, which distributes responsibility for bugs and vulnerabilities. External contributions go through maintainer review, creating a merit-based filter. Forks let projects follow different paths when visions diverge, without losing accumulated knowledge.

That model produced critical infrastructure that private companies would not have built the same way: without dependence on a single vendor, without license cost, without access barriers for researchers and developers anywhere in the world.

## Beyond software

The open source principle migrated to other fields, with concrete results.

**Hardware**: Arduino made electronics development accessible to hobbyists, researchers, and artists. The RISC-V architecture is an open instruction set that competes with ARM and x86 in research and embedded devices. The Open Compute Project, started by Facebook, opened data center, server, and power system designs.

**Science and research**: OSGeo (Open Source Geospatial Foundation) maintains tools such as QGIS, GDAL, and PostGIS, widely used in environmental monitoring, urban planning, and remote sensing. The Open Energy Modelling Initiative applies the same principle to energy models used in research and public policy.

**Agriculture and ecology**: Open Source Ecology developed the Global Village Construction Set, a set of 50 modular machines with fully open designs. OpenSourceSeeds applies copyleft to seeds, preventing collectively developed varieties from being patented and removed from the public domain.

**Medicine**: Open pharmaceutical development initiatives led to consortia focused on neglected diseases such as malaria. In bioinformatics, rapid sharing of sequencing data during the 2011 E. coli outbreak showed how the model accelerates crisis response.

The pattern repeats: where there is a common good that no single actor has an incentive to build alone, the open model tends to emerge as a viable alternative.

## Instituto Forest's commitment

Instituto Forest was born in this context and does not pretend to be neutral toward it. Our data pipelines, models, and documentation are published as open source not as a communications differentiator, but as a working method: auditability, reproducibility, and continuous contribution are part of what it means to work seriously with environmental data.

We believe the open source community is a common good that must be fed, not only consumed. That means accepting contributions, keeping documentation alive, and using open tools whenever possible. It also means recognizing that the public data we process belongs to society, and that making our work reproducible is an obligation, not a concession.
